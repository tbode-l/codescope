# 组员补充清单

目前先忽略代码里的运行问题，只记录需要组员补充到上游数据里的字段。

## 需要补到 `known_competitors.json` / `analysis.json` 的字段

1. `forks`
   - 来源：GitHub 仓库元数据 `forks_count`
   - 用途：作为关注度评分的辅助指标

2. `open_issues` 或 `issues`
   - 来源：GitHub 仓库元数据 `open_issues_count`
   - 用途：作为项目活跃互动信号

3. `updated_at`
   - 来源：GitHub 仓库元数据 `updated_at`
   - 用途：判断项目是否近期维护，修正关注度和技术先进性评分

4. `readme_path`
   - 来源：`tracker.py` 下载 README 后生成的本地路径
   - 用途：让 `scorer.py` 必要时可以读取原始 README，补充判断 API、插件、SDK、示例等信号

## 可选增强字段

如果组员愿意在 `analyzer.py` 里多提取一些布尔字段，评分会更准：

1. `has_api`
2. `has_cli`
3. `has_plugins`
4. `has_sdk`
5. `has_examples`
6. `has_hooks`
7. `has_quick_start`
8. `supports_one_command_run`

这些字段不是必须，但可以增强“生态开放性”和“配置门槛与产品形态”的评分准确度。
